## Navigation de base

### Théorie : Se déplacer dans l'arborescence

Le terminal Linux utilise une structure arborescente pour organiser les fichiers et dossiers. Chaque utilisateur a un répertoire personnel et peut naviguer dans tout le système selon ses permissions.

### Commandes essentielles

#### Commandes de navigation

| Commande | Description |
|----------|-------------|
| **pwd** | Affiche le répertoire actuel |
| **ls** | Liste le contenu du répertoire |
| **ls -la** | Liste détaillée avec fichiers cachés |
| **cd** | Change de répertoire |
| **cd ..** | Remonte d'un niveau |
| **cd ~** | Va au répertoire personnel |

#### Raccourcis utiles

| Raccourci | Description |
|-----------|-------------|
| **Tab** | Autocomplétion |
| **↑ ↓** | Historique des commandes |
| **Ctrl+C** | Interrompre commande |
| **Ctrl+L** | Nettoyer l'écran |
| **~** | Répertoire personnel |
| **.** | Répertoire actuel |

### Exemples pratiques

**Scénario : Organiser un projet de classification d'images**

1. **pwd** - Vérifier où vous êtes
2. **cd ~** - Aller au répertoire personnel
3. **ls -la** - Voir ce qui existe déjà
4. **cd projets_ia** - Aller dans le dossier projets (s'il existe)
5. **ls** - Lister les projets existants
6. **cd classification_images** - Entrer dans le projet
7. **pwd** - Confirmer la localisation

## Permissions et propriétés

### Théorie : Système de permissions Linux

Linux utilise un système de permissions basé sur trois niveaux : propriétaire (user), groupe (group), et autres (other). Chaque fichier/dossier a des permissions de lecture (r), écriture (w), et exécution (x).

### Comprendre les permissions

#### Format d'affichage des permissions

Exemple : `-rwxr-xr--`
- **1er caractère** : Type (- fichier, d dossier, l lien)
- **Caractères 2-4** : Permissions propriétaire (rwx)
- **Caractères 5-7** : Permissions groupe (r-x)
- **Caractères 8-10** : Permissions autres (r--)

#### Commandes de gestion des permissions

| Commande | Description |
|----------|-------------|
| **ls -l** | Affiche les permissions détaillées |
| **chmod** | Modifie les permissions |
| **chown** | Change le propriétaire |
| **chgrp** | Change le groupe |

### Exemples pratiques

**Scénario : Sécuriser des scripts et données IA**

#### Permissions typiques pour l'IA

| Type de fichier | Permissions recommandées | Exemple |
|-----------------|--------------------------|---------|
| **Scripts Python** | rwxr-xr-x (755) | Scripts exécutables par tous |
| **Données sensibles** | rw------- (600) | Datasets privés |
| **Modèles entraînés** | rw-r--r-- (644) | Modèles lisibles par l'équipe |
| **Notebooks** | rw-rw-r-- (664) | Notebooks collaboratifs |

#### Notation octale des permissions

| Octal | Binaire | Permissions | Description |
|-------|---------|-------------|-------------|
| 7 | 111 | rwx | Lecture, écriture, exécution |
| 6 | 110 | rw- | Lecture, écriture |
| 5 | 101 | r-x | Lecture, exécution |
| 4 | 100 | r-- | Lecture seule |
| 0 | 000 | --- | Aucune permission |

## Recherche et filtrage

### Théorie : Localiser efficacement fichiers et contenu

En IA, vous devez souvent localiser des fichiers spécifiques, rechercher dans des logs, ou filtrer des résultats. Les commandes de recherche sont essentielles pour la productivité.

### Commandes de recherche de fichiers

#### find - Recherche avancée de fichiers

| Option | Description |
|--------|-------------|
| **find . -name "*.py"** | Trouve tous les fichiers Python |
| **find . -type d** | Trouve tous les dossiers |
| **find . -size +100M** | Trouve fichiers > 100MB |
| **find . -mtime -7** | Fichiers modifiés cette semaine |

#### locate - Recherche rapide par nom

| Commande | Description |
|----------|-------------|
| **locate dataset** | Trouve tous les fichiers contenant "dataset" |
| **updatedb** | Met à jour la base de données locate |

### Commandes de recherche de contenu

#### grep - Recherche dans le contenu

| Option | Description |
|--------|-------------|
| **grep "error" *.log** | Cherche "error" dans les logs |
| **grep -r "import torch"** | Recherche récursive |
| **grep -i "tensorflow"** | Recherche insensible à la casse |
| **grep -n "def main"** | Affiche les numéros de ligne |

#### Filtrage et combinaisons

| Commande | Description |
|----------|-------------|
| **ls \| grep .py** | Liste uniquement les fichiers Python |
| **find . -name "*.csv" \| head -10** | 10 premiers CSV trouvés |
